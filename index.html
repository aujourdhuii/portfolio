PROJECT 01

아라폼(AraForm) 내부관리자 페이지 기획

신규 서비스 기획 플랫폼 백오피스 Figma 신사업 TF GitBook
⚠️
Problem
  • 진학사 신규 사업인 아라폼(AraForm) 출시 준비 단계에서 내부관리자 페이지가 없는 상태
  • 기관 관리, 회원 관리, 금액권 관리 등 운영에 필요한 내부관리자 기능 부재로 서비스 운영 불가
🔧
Solution
  • 운영팀에서 TF로 합류하여 신규 서비스 런칭 준비 단계부터 기획 주도
  • Figma로 내부관리자 페이지 28개 화면 설계 및 개발팀 협업으로 구현
  • 기관별 서비스 사용 현황, 회원 관리, 금액권 관리 등 운영 핵심 기능 기획
  • GitBook 기반 아라폼 서비스 가이드 제작으로 운영 정보 관리 체계 완성
  • 실제 운영 경험을 바탕으로 현장 흐름에 맞는 UX 설계
📈
Impact
28개 화면 설계 완료
  • 내부관리자 페이지 신규 설계로 아라폼 정식 출시 기반 마련
  • 운영팀 관점의 기획으로 실무 친화적 백오피스 완성
  • GitBook 가이드 제작으로 서비스 운영 정보 체계화
PROJECT 02

실적증명원 통합관리 시스템

신규 서비스 기획 데이터 기반 설계 API 연동 운영 자동화 Supabase Vercel
⚠️
Problem
  • 엑셀 수기 관리로 등기 등록 건당 약 50초 소요 (매년 300건+) — 반복적 비효율 구조 고착
  • 도착/검수 상태 수기 관리로 누락·오류 반복, 데이터 신뢰도 저하
  • 실적 데이터 분산으로 성과 비교·분석 불가 → 의사결정 지연
  • 핵심 인사이트: 입력·조회·상태관리를 시스템화하면 오류를 줄이고 데이터 기반 의사결정이 가능해진다
🔧
Solution
  • 웹 기반 통합관리 서비스 기획 및 구축, Vercel 배포로 사내 접근 환경 구성
  • Supabase(PostgreSQL) 기반 DB 설계로 실시간 데이터 관리 및 권한 제어
  • 바코드 스캐너 + 우체국 API 연동으로 수기 입력 완전 제거
  • 바코드 등록 시 운영자 자동 매칭 및 상태 자동 반영 (State Automation)
  • 담당자별 대학·접수 유형 필터링 및 도착·검수율 퍼센테이지 시각화
  • 학년도별 실적 대시보드로 데이터 기반 성과 비교 및 영업 전략 수립 지원
📈
Impact
90% 처리시간 단축
−125분 연간 절감 (300건 기준)
  • 등기 등록 시간 50초 → 5초
  • 수기 입력 제거 및 데이터 오류 감소
  • 운영 프로세스 단순화 및 표준화
  • 데이터 기반 영업 의사결정 지원
PROJECT 03

보증보험 자동 알림 시스템

서비스 자동화 기획 리스크 관리 Microsoft Graph API 운영 안정성
⚠️
Problem
  • 보험 만료일 수기 확인 후 담당자가 직접 공지 — 사람에 의존한 구조
  • 알림 누락 시 계약 공백 등 직접적 업무 리스크 발생 가능
  • 핵심 인사이트: 알림을 시스템화하면 리스크를 제거하고 운영 신뢰도를 높일 수 있다
🔧
Solution
  • SharePoint Excel과 Microsoft Graph API, Teams 연동으로 데이터 소스 자동 연결
  • 만료 20일 전 자동 메일 발송 로직 설계 및 구현
  • 발송 이력 Excel 자동 기록으로 감사 추적(Audit Trail) 가능하도록 설계
📈
Impact
0건 알림 누락
100% 자동화 달성
  • 반복 업무 완전 자동화로 운영 효율 향상
  • 알림 누락 리스크 구조적으로 제거
  • 발송 이력 자동 기록으로 운영 안정성 확보
PROJECT 04

대학 관리자 서비스 개선

플랫폼 UX 개선 사용자 경험 분석 서비스 고도화 Figma ChatGPT
⚠️
Problem
  • 대학별로 각각 존재하던 지원자 기재사항 페이지로 인해 관리 파편화 및 운영 비효율 발생
  • 관리자별 시스템 이해도 차이로 설정 오류·문의·수정 요청 지속 증가
  • 핵심 인사이트: 기재사항 페이지를 하나의 표준으로 통합하면 문의가 줄고 운영 효율이 개선된다
🔧
Solution
  • 대학별로 분산된 기재사항 페이지를 하나의 표준화된 구조로 통합 기획
  • 관리자 페이지 사용자 흐름 분석 및 문제 구간 특정 후 설정 방식 개선
  • 기획 도구 PPT → Figma 전환으로 개선안 시각화 및 개발팀 협업 효율 향상
  • 수정요청 프로세스 구조화 및 사용자 관점의 커뮤니케이션 가이드 수립
  • ChatGPT 활용해 매뉴얼 전면 업데이트 및 AI 기반 문서 표준화로 반복 업무 효율 개선
📈
Impact
↓ 문의·수정 요청 감소
↑ 운영 효율화
  • 기재사항 페이지 표준화로 대학별 설정 오류 및 문의 건수 감소
  • Figma·AI 기반 기획 프로세스 표준화로 팀 생산성 향상
  • 운영 구조 단순화로 서비스 안정성 및 운영 효율화 확보
